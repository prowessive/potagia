## Project Guidelines

### Language
- All source code must be written in English.
- All code comments must be written in English.
- All project documentation must be written in English.
- When editing an existing file with non-English text, translate the edited content to English.

### Design and Code Quality
- Prefer Object-Oriented Programming (OOP) where it fits the project architecture.
- Follow Clean Code principles:
  - Use meaningful and intention-revealing names.
  - Keep functions and methods small and focused on a single responsibility.
  - Avoid duplicated logic (DRY).
  - Keep modules cohesive and dependencies explicit.
  - Write readable code first; avoid unnecessary complexity.
